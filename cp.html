문제 1. 포인터 선언과 역참조

#include <stdio.h>
int main() {
    int a = 10;
    int *p = &a;
    printf("%d\n", *p);
    return 0;
}
예상 결과: 10
상세 설명: p에는 변수 a의 주소가 저장됩니다. *p는 그 주소에 저장된 실제 값을 뜻하므로 10이 출력됩니다.
시험 포인트: &a는 주소, *p는 값입니다.

문제 2. 주소 출력

#include <stdio.h>
int main() {
    int a = 25;
    int *p = &a;
    printf("%p %p\n", (void*)&a, (void*)p);
    return 0;
}
예상 결과: 두 값이 같은 주소로 출력됨
상세 설명: &a는 변수 a의 주소이고, p에도 같은 주소가 저장되어 있습니다. 따라서 두 출력값은 같습니다. 주소는 실행할 때마다 달라질 수 있습니다.
시험 포인트: 주소 출력에는 보통 %p를 사용하고, 관례적으로 (void*)로 형변환합니다.

문제 3. 포인터로 값 변경

#include <stdio.h>
int main() {
    int a = 7;
    int *p = &a;
    *p = 99;
    printf("%d\n", a);
    return 0;
}
예상 결과: 99
상세 설명: *p = 99;는 포인터가 가리키는 공간에 99를 저장하라는 뜻입니다. p는 a를 가리키므로 결국 a의 값이 바뀝니다.
시험 포인트: 포인터를 사용하면 원본 변수를 직접 수정할 수 있습니다.

문제 4. 두 변수와 두 포인터

#include <stdio.h>
int main() {
    int x = 3, y = 8;
    int *px = &x, *py = &y;
    printf("%d %d\n", *px, *py);
    return 0;
}
예상 결과: 3 8
상세 설명: 각각의 포인터가 서로 다른 변수를 가리킵니다. *px는 x, *py는 y의 값을 읽습니다.
시험 포인트: 포인터가 어느 변수를 가리키는지 정확히 연결해서 해석해야 합니다.

문제 5. 포인터 대입 방향

#include <stdio.h>
int main() {
    int a = 5;
    int b = 12;
    int *p = &a;
    p = &b;
    printf("%d\n", *p);
    return 0;
}
예상 결과: 12
상세 설명: 처음에는 p가 a를 가리키지만, p = &b; 이후에는 b를 가리킵니다. 따라서 *p는 12입니다.
시험 포인트: 포인터 변수 자체의 값은 "주소"이며, 다른 주소를 대입하면 가리키는 대상이 바뀝니다.

문제 6. 함수와 주소 전달

#include <stdio.h>
void change(int *p) {
    *p = 50;
}
int main() {
    int a = 10;
    change(&a);
    printf("%d\n", a);
    return 0;
}
예상 결과: 50
상세 설명: 함수로 변수의 주소를 넘기면 함수 안에서 원본 값을 바꿀 수 있습니다. change(&a)는 a의 주소를 전달하고, 함수 안의 *p = 50;이 원본을 변경합니다.
시험 포인트: C는 기본적으로 값 복사 호출이므로, 원본 변경이 필요하면 주소를 넘겨야 합니다.

문제 7. swap 함수

#include <stdio.h>
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}
int main() {
    int x = 2, y = 9;
    swap(&x, &y);
    printf("%d %d\n", x, y);
    return 0;
}
예상 결과: 9 2
상세 설명: 포인터 매개변수로 두 변수의 주소를 받아 서로의 값을 교환합니다. 이 패턴은 포인터 활용의 대표 예시입니다.
시험 포인트: 값 자체가 아니라 주소를 주고받기 때문에 함수 밖 변수도 바뀝니다.

문제 8. 배열 이름과 첫 번째 원소

#include <stdio.h>
int main() {
    int arr[3] = {10, 20, 30};
    printf("%d %d\n", arr[0], *arr);
    return 0;
}
예상 결과: 10 10
상세 설명: 배열 이름 arr은 대부분의 표현식에서 첫 번째 원소의 주소처럼 동작합니다. 따라서 *arr는 arr[0]와 같습니다.
시험 포인트: arr == &arr[0]처럼 이해하면 편합니다.

문제 9. 포인터 연산

#include <stdio.h>
int main() {
    int arr[3] = {10, 20, 30};
    int *p = arr;
    p++;
    printf("%d\n", *p);
    return 0;
}
예상 결과: 20
상세 설명: p가 처음에는 arr[0]을 가리키고, p++ 후에는 다음 원소인 arr[1]을 가리킵니다. 그래서 *p는 20입니다.
시험 포인트: 포인터 연산은 자료형 크기만큼 이동합니다. int*라면 보통 4바이트 단위로 이동합니다.

문제 10. 배열 순회

#include <stdio.h>
int main() {
    int arr[4] = {1, 2, 3, 4};
    int *p = arr;
    for (int i = 0; i < 4; i++) {
        printf("%d ", *(p + i));
    }
    return 0;
}
예상 결과: 1 2 3 4
상세 설명: p + i는 배열의 i번째 원소 위치로 이동한 주소입니다. 여기에 *를 붙이면 그 위치의 값을 읽습니다.
시험 포인트: arr[i]와 *(arr + i)는 같습니다.

문제 11. 문자열 포인터

#include <stdio.h>
int main() {
    char *s = "ABC";
    printf("%c %c\n", *s, *(s + 1));
    return 0;
}
예상 결과: A B
상세 설명: 문자열 리터럴의 시작 주소가 s에 저장됩니다. *s는 첫 글자 'A', *(s+1)는 두 번째 글자 'B'입니다.
시험 포인트: 문자열도 결국 문자 배열처럼 연속된 메모리입니다.

문제 12. 문자 배열 수정

#include <stdio.h>
int main() {
    char s[] = "ABC";
    char *p = s;
    *(p + 1) = 'Z';
    printf("%s\n", s);
    return 0;
}
예상 결과: AZC
상세 설명: 이번에는 문자열 리터럴이 아니라 수정 가능한 문자 배열입니다. 두 번째 문자 위치에 'Z'를 넣었으므로 결과는 AZC입니다.
시험 포인트: char *s = "ABC";와 char s[] = "ABC";는 수정 가능성에서 차이가 큽니다.

문제 13. const 포인터 기초

#include <stdio.h>
int main() {
    int a = 10;
    const int *p = &a;
    printf("%d\n", *p);
    return 0;
}
예상 결과: 10
상세 설명: const int *p는 "포인터를 통해 값을 바꾸지 않겠다"는 뜻입니다. 읽기는 가능하지만 *p = 20; 같은 수정은 허용되지 않습니다.
시험 포인트: const가 무엇을 제한하는지 해석하는 문제가 자주 나옵니다.

문제 14. 포인터 배열

#include <stdio.h>
int main() {
    char *words[3] = {"cat", "dog", "bird"};
    printf("%s %c\n", words[1], *words[2]);
    return 0;
}
예상 결과: dog b
상세 설명: words는 문자열 주소를 저장하는 포인터들의 배열입니다. words[1]은 "dog", *words[2]는 "bird"의 첫 글자 'b'입니다.
시험 포인트: "배열의 각 칸에 주소가 들어 있다"고 생각하면 구조가 잘 보입니다.

문제 15. 이중 포인터

#include <stdio.h>
int main() {
    int a = 5;
    int *p = &a;
    int **pp = &p;
    printf("%d\n", **pp);
    return 0;
}
예상 결과: 5
상세 설명: pp는 포인터 p의 주소를 저장합니다. 따라서 *pp는 p, **pp는 결국 a의 값입니다.
시험 포인트: 별표 개수만큼 단계적으로 따라가며 해석하면 됩니다.

문제 16. malloc 기초

#include <stdio.h>
#include <stdlib.h>
int main() {
    int *p = (int*)malloc(sizeof(int));
    *p = 77;
    printf("%d\n", *p);
    free(p);
    return 0;
}
예상 결과: 77
상세 설명: malloc으로 int 1개를 저장할 메모리를 동적으로 확보합니다. 그 공간에 77을 저장하고 출력한 뒤, 마지막에 free로 반납합니다.
시험 포인트: 동적 할당 후에는 사용이 끝나면 반드시 free를 고려해야 합니다.

문제 17. 동적 배열 접근

#include <stdio.h>
#include <stdlib.h>
int main() {
    int *p = (int*)malloc(sizeof(int) * 3);
    for (int i = 0; i < 3; i++) p[i] = (i + 1) * 10;
    printf("%d %d %d\n", *p, *(p + 1), p[2]);
    free(p);
    return 0;
}
예상 결과: 10 20 30
상세 설명: 동적 메모리도 연속 공간이므로 배열처럼 사용할 수 있습니다. p[i], *(p+i), *p는 모두 주소 계산을 바탕으로 동작합니다.
시험 포인트: 포인터와 배열 표기법은 매우 밀접합니다.

문제 18. 함수에서 포인터 반환 주의

#include <stdio.h>
int* bad() {
    int x = 10;
    return &x;
}
int main() {
    int *p = bad();
    printf("%p\n", (void*)p);
    return 0;
}
예상 결과: 컴파일은 될 수 있으나 매우 위험한 코드
상세 설명: 함수 안의 지역변수 x는 함수가 끝나면 사라집니다. 그런데 그 주소를 반환하면, 이미 유효하지 않은 메모리를 가리키는 포인터가 됩니다. 이런 포인터를 댕글링 포인터라고 합니다.
시험 포인트: 지역변수의 주소를 반환하면 안 됩니다.

문제 19. NULL 포인터

#include <stdio.h>
int main() {
    int *p = NULL;
    if (p == NULL)
        printf("NULL pointer\n");
    return 0;
}
예상 결과: NULL pointer
상세 설명: NULL은 아무것도 가리키지 않는 포인터를 뜻합니다. 역참조하면 안 되며, 먼저 NULL인지 확인하는 습관이 중요합니다.
시험 포인트: *p를 하기 전에 p가 유효한 주소인지 확인해야 합니다.

문제 20. 종합 문제

#include <stdio.h>
void addOne(int *p) {
    (*p)++;
}
int main() {
    int arr[3] = {2, 4, 6};
    int *p = arr;
    addOne(p + 1);
    printf("%d %d %d\n", arr[0], arr[1], arr[2]);
    return 0;
}
예상 결과: 2 5 6
상세 설명: p+1은 두 번째 원소 arr[1]의 주소입니다. 이를 함수에 넘기면 함수 안의 (*p)++;가 그 원소만 1 증가시킵니다. 따라서 4가 5가 됩니다.
시험 포인트: 배열 + 포인터 연산 + 함수 호출이 함께 섞여 있는 전형적인 응용 문제입니다.

최종 핵심 정리

  • & : 변수의 주소를 구한다.
  • * : 포인터 선언에 쓰이거나, 주소를 따라가 실제 값을 읽고 쓴다.
  • 배열 이름은 대부분 첫 번째 원소의 주소처럼 동작한다.
  • 원본 값을 바꾸려면 함수에 주소를 넘겨야 한다.
  • NULL, 댕글링 포인터, 잘못된 역참조는 반드시 주의해야 한다.

학습 팁: 포인터 문제는 항상 “지금 이 포인터가 누구의 주소를 들고 있는가?”를 먼저 따지면 훨씬 쉽게 풀립니다.
