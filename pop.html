리스트, 딕셔너리, 스택, 큐, 반복문, 조건문, 중첩 리스트까지 pop()의 핵심 동작을 코드와 문제로 완전 정리합니다.

핵심 개념 정리

list.pop()
기본값은 마지막 원소 제거 후 반환
list.pop(index)
해당 인덱스의 원소를 제거하고 반환
dict.pop(key)
해당 key의 value를 제거하고 반환
dict.pop(key, default)
key가 없으면 오류 대신 default 반환
표현의미주의점
a.pop()리스트 마지막 원소 제거 및 반환빈 리스트면 오류
a.pop(0)리스트 첫 원소 제거 및 반환큰 리스트에서는 느릴 수 있음
d.pop("x")딕셔너리 key "x" 제거 및 value 반환key가 없으면 오류

01. 리스트 마지막 원소 꺼내기

list 기본 pop
개념 설명: pop()은 리스트의 마지막 원소를 제거하면서 그 값을 반환한다.

파이썬 코드

a = [10, 20, 30, 40]
x = a.pop()

print(x)
print(a)

실행 결과

40 [10, 20, 30]

고난도 5지선다 문제 1

다음 코드 실행 후 변수 x와 리스트 a의 값으로 옳은 것은?

  1. x = 10, a = [20, 30, 40]
  2. x = 40, a = [10, 20, 30]
  3. x = None, a = [10, 20, 30]
  4. x = 40, a = [10, 20, 30, 40]
  5. 오류가 발생한다
정답: ②
상세 해설: pop()은 삭제된 값을 반환하므로 x는 40이고, 리스트에서는 40이 제거된다.

02. 특정 인덱스 원소 꺼내기

list.pop(index)
개념 설명: pop(1)은 인덱스 1의 원소를 제거한다. 리스트 인덱스는 0부터 시작한다.

파이썬 코드

a = ['A', 'B', 'C', 'D']
x = a.pop(1)

print(x)
print(a)

실행 결과

B ['A', 'C', 'D']

고난도 5지선다 문제 2

위 코드의 출력 결과로 옳은 것은?

  1. A와 ['B','C','D']
  2. B와 ['A','C','D']
  3. C와 ['A','B','D']
  4. D와 ['A','B','C']
  5. IndexError
정답: ②
상세 해설: 인덱스 1은 'B'이다. 따라서 'B'가 반환되고 리스트에서는 삭제된다.

03. 음수 인덱스 pop

negative index
개념 설명: 음수 인덱스는 뒤에서부터 센다. -1은 마지막, -2는 뒤에서 두 번째이다.

파이썬 코드

print(x)
print(a)

실행 결과

  1. 10
  2. 15
  3. 20
  4. 25
  5. IndexError
정답: ③
상세 해설: -2는 뒤에서 두 번째 원소이므로 20이 제거되고 반환된다.

04. while문으로 역순 출력

stack behavior
개념 설명: 리스트의 끝에서 계속 pop하면 나중에 들어간 값부터 나온다. 이것이 스택의 LIFO 구조이다.

파이썬 코드

while a:
    print(a.pop())

실행 결과

  4. 4 2 3 1
  5. 무한 반복
정답: ②
상세 해설: pop()은 매번 마지막 원소를 꺼내므로 4, 3, 2, 1 순서로 출력된다.

append + pop

20. pop으로 괄호 검사

stack algorithm
개념 설명: 여는 괄호는 stack에 넣고 닫는 괄호가 나오면 pop한다. 마지막에 stack이 비어 있으면 괄호가 맞다.

파이썬 코드

text = "(()())"
stack = []

ok = True
for ch in text:
    if ch == '(':
        stack.append(ch)
    elif ch == ')':
        if stack:
            stack.pop()
        else:
            ok = False

print(ok and not stack)

실행 결과

True

고난도 5지선다 문제 20

출력이 True인 이유로 가장 적절한 것은?

  1. 닫는 괄호가 하나도 없기 때문
  2. 여는 괄호가 더 많기 때문
  3. 모든 닫는 괄호가 이전 여는 괄호와 짝을 이루고 마지막 stack이 비었기 때문
  4. pop이 항상 True를 반환하기 때문
  5. 문자열이 자동 정렬되기 때문
정답: ③
상세 해설: 각 닫는 괄호마다 대응하는 여는 괄호가 pop되고, 끝에 남은 여는 괄호가 없으므로 True이다.

마무리 핵심

  • pop()은 삭제와 반환을 동시에 수행한다.
  • 리스트에서 pop()은 마지막 원소, pop(i)는 인덱스 i의 원소를 다룬다.
  • 딕셔너리에서 pop(key)는 key에 해당하는 value를 반환하고 해당 항목을 삭제한다.
  • 빈 리스트에서 pop()은 IndexError, 없는 딕셔너리 key를 기본값 없이 pop하면 KeyError가 발생한다.
