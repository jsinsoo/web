컴퓨팅 시스템

12정01-01 유무선 네트워크의 특성을 이해하고, 컴퓨팅 시스템 간 공유, 협력, 소통을 위한 네트워크 환경을 구성한다.

수행활동 예시

교실/가정 네트워크 구성도 설계 및 공유 폴더·접속 권한 설정 실습

  • 문제 상황 또는 자료를 분석한다.
  • 핵심 개념을 적용해 산출물을 만든다.
  • 결과를 발표하고 한계와 개선 방향을 정리한다.

평가 요소

네트워크 장비와 유무선 특성을 비교하고, 실제 협업 목적에 맞는 네트워크 구성·공유 설정안을 작성한다.

과세특 기록 예시

상: 학교 무선망 접속 지연 문제를 사례로 삼아 AP, 공유기, 스위치, 단말의 역할을 실제 배치도에 표시하고, 협업 수업을 위한 공유 폴더 권한을 읽기·쓰기·관리자로 세분화함. 유선은 안정성과 속도, 무선은 이동성과 확장성이 강점이라는 점을 근거 자료와 함께 설명하였으며, 개인정보가 포함된 산출물은 제한 공유로 설정해야 한다는 보안 관점까지 반영함. 네트워크 구성 결과를 발표할 때 “접속은 쉽지만 권한은 최소화한다”는 원칙을 스스로 도출한 점이 인상적임.
중: 교실 네트워크 구성도 작성 활동에서 유무선 네트워크의 차이를 이해하고 공유 폴더 사용 목적에 맞게 접속 권한을 설정함. 처음에는 모든 사용자를 동일 권한으로 두었으나 피드백 후 발표자, 편집자, 열람자를 구분하여 수정하였고, 협업 상황에서 네트워크 환경이 학습 결과물의 공유 방식에 영향을 준다는 점을 설명함.
하: 네트워크 장비의 명칭과 기능을 예시 자료를 참고하여 정리하고, 교실 네트워크 구성도에 일부 반영함. 공유 권한 설정 과정에서 읽기와 쓰기 권한의 차이를 헷갈려 했으나 교사의 안내를 받아 수정하였으며, 협력 활동에서 안전한 공유 설정이 필요하다는 점을 인식함.

devices = ["교사용PC","학생노트북","프린터","NAS","스마트폰"]
connections = {
    "공유기": ["교사용PC","학생노트북","프린터","NAS","스마트폰"],
    "NAS": ["수업자료","제출함"],
    "프린터": ["교사용PC","학생노트북"]
}
for hub, nodes in connections.items():
    print(hub, "->", ", ".join(nodes))

상: 문자열 압축 실험에서 반복 문자가 많은 데이터와 불규칙한 데이터의 압축률을 직접 계산하고, 압축 효율이 데이터의 패턴성에 따라 달라진다는 결론을 도출함. 원본 크기, 압축 후 크기, 압축률을 표와 그래프로 제시하였으며, “무조건 압축하면 작아진다”는 오개념을 실험 결과로 반박함. 이미지·음악·문서 파일에서 손실/무손실 압축을 구분하여 설명한 점이 우수함.
중: 간단한 문자열 압축 코드를 실행하여 원본과 압축 결과의 길이를 비교함. 반복 문자가 많을수록 압축 효과가 커진다는 사실을 확인하고, 압축률 계산식을 적용하여 결과를 정리함. 압축이 저장 공간과 전송 시간 절약에 필요하다는 점을 이해함.
하: 문자열 압축 예시를 따라 실행하고 원본 길이와 압축 길이를 비교함. 압축률 계산 과정에서 일부 오류가 있었으나 피드백 후 수정하였으며, 데이터 압축이 파일 크기를 줄이기 위한 방법임을 설명할 수 있음.

파이썬 코드 예시

text = "AAAABBBCCDAA"
compressed = []
count = 1
for i in range(1, len(text)+1):
    if i < len(text) and text[i] == text[i-1]:
        count += 1
    else:
        compressed.append(text[i-1] + str(count))
        count = 1
result = "".join(compressed)
print("원본:", text, len(text))
print("압축:", result, len(result))
print("압축률:", round(len(result)/len(text), 2))

def selection_sort(a):
    count = 0
    for i in range(len(a)):
        m = i
        for j in range(i+1, len(a)):
            count += 1
            if a[j] < a[m]:
                m = j
        a[i], a[m] = a[m], a[i]
    return a, count

def binary_search(data, target):
    left, right = 0, len(data)-1
    count = 0
    while left <= right:
        count += 1
        mid = (left + right) // 2
        if data[mid] == target:
            return mid, count
        elif data[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1, count
print(binary_search([1,3,5,7,9,11,13], 11))
