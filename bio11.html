개념 1

세포설

① 핵심 개념
모든 생물은 세포로 이루어져 있으며, 세포는 생명 활동의 기본 단위이다.
② 개념 상세 설명
  1. 세포는 생명체를 구성하는 가장 기본적인 구조 단위이다.
  2. 모든 생명체는 하나 이상의 세포로 이루어져 있다.
  3. 새로운 세포는 기존 세포의 분열로 생긴다.
  4. 세포 안에서는 물질대사, 에너지 생산, 유전 정보 전달이 일어난다.
  5. 세포설은 현대 생물학의 출발점이 되는 핵심 원리이다.
③ 활용 예제
현미경 관찰, 조직 연구, 의학 진단, 세포 배양 실험에 활용된다.
④ 파이썬 코드
# 세포 수 증가 모델: 세포가 한 번 분열할 때마다 2배 증가
cells = 1
divisions = 10

for i in range(divisions):
    cells *= 2

print("분열 후 세포 수:", cells)
⑤ 코드 상세 설명
  1. `cells = 1` : 처음 세포 수를 1개로 둔다.
  2. `divisions = 10` : 세포 분열 횟수이다.
  3. `for i in range(divisions)` : 분열 횟수만큼 반복한다.
  4. `cells *= 2` : 한 번 분열할 때마다 세포 수가 2배가 된다.
  5. 출력값은 10번 분열 후 세포 수이다.
개념 2

DNA와 유전 정보

① 핵심 개념
DNA는 염기서열 형태로 유전 정보를 저장하며, 염기는 \(A,T,G,C\) 네 종류로 표현된다.
② 개념 상세 설명
  1. DNA는 생명체의 유전 정보를 담고 있는 분자이다.
  2. DNA의 기본 단위는 뉴클레오타이드이다.
  3. 염기 A는 T와, G는 C와 상보적으로 결합한다.
  4. DNA 염기서열은 단백질 합성 정보를 담고 있다.
  5. DNA 정보는 복제와 전사를 통해 다음 세대와 세포 활동에 전달된다.
③ 활용 예제
유전자 분석, 친자 확인, 질병 진단, 생명공학, 법의학에 활용된다.
④ 파이썬 코드
# DNA 염기 개수 세기
dna = "ATGCGATACGTT"

counts = {}
for base in dna:
    counts[base] = counts.get(base, 0) + 1

print(counts)
⑤ 코드 상세 설명
  1. `dna` : DNA 염기서열을 문자열로 저장한다.
  2. `counts = {}` : 염기 개수를 저장할 딕셔너리이다.
  3. `for base in dna` : 염기를 하나씩 확인한다.
  4. `counts.get(base, 0)` : 해당 염기가 처음 나오면 0부터 시작한다.
  5. 각 염기의 등장 횟수를 계산한다.
개념 3

상보적 염기쌍

① 핵심 개념
DNA에서 아데닌 \(A\)은 티민 \(T\)과, 구아닌 \(G\)은 사이토신 \(C\)과 결합한다.
② 개념 상세 설명
  1. 상보적 염기쌍은 DNA 이중나선 구조의 핵심이다.
  2. A와 T는 서로 결합하고, G와 C는 서로 결합한다.
  3. 이 규칙 덕분에 DNA는 정확하게 복제될 수 있다.
  4. 한 가닥의 염기서열을 알면 반대 가닥을 예측할 수 있다.
  5. 염기쌍 오류는 돌연변이의 원인이 될 수 있다.
③ 활용 예제
DNA 복제, PCR, 유전자 서열 분석, 생명공학 실험에 활용된다.
④ 파이썬 코드
# DNA 상보적 가닥 만들기
dna = "ATGCCGTA"
pair = {"A": "T", "T": "A", "G": "C", "C": "G"}

complement = "".join(pair[base] for base in dna)

print("원래 가닥:", dna)
print("상보 가닥:", complement)
⑤ 코드 상세 설명
  1. `pair` : 각 염기의 상보 염기를 저장한 딕셔너리이다.
  2. `for base in dna` : DNA 문자열의 염기를 하나씩 꺼낸다.
  3. `pair[base]` : 현재 염기의 상보 염기를 찾는다.
  4. `''.join(...)` : 상보 염기들을 하나의 문자열로 합친다.
  5. 결과는 원래 DNA 가닥의 상보 가닥이다.
개념 4

단백질 합성

① 핵심 개념
DNA의 정보는 RNA로 전사되고, RNA의 코돈은 아미노산으로 번역되어 단백질을 만든다.
② 개념 상세 설명
  1. 단백질 합성은 전사와 번역으로 이루어진다.
  2. 전사는 DNA 정보를 mRNA로 옮기는 과정이다.
  3. 번역은 mRNA의 코돈을 아미노산 서열로 바꾸는 과정이다.
  4. 코돈은 염기 3개로 이루어진 유전 암호이다.
  5. 단백질은 효소, 구조 단백질, 호르몬 등 다양한 기능을 수행한다.
③ 활용 예제
유전자 발현, 단백질 공학, 질병 원인 분석, 생명공학에 활용된다.
④ 파이썬 코드
# 간단한 코돈 번역 예시
rna = "AUGUUUUGA"
codon_table = {"AUG": "Met", "UUU": "Phe", "UGA": "Stop"}

protein = []
for i in range(0, len(rna), 3):
    codon = rna[i:i+3]
    protein.append(codon_table.get(codon, "?"))

print(protein)
⑤ 코드 상세 설명
  1. `rna` : mRNA 염기서열이다.
  2. `codon_table` : 코돈과 아미노산 대응표이다.
  3. `range(0, len(rna), 3)` : 염기 3개씩 읽는다.
  4. `rna[i:i+3]` : 현재 코돈을 자른다.
  5. `codon_table.get(codon, '?')` : 코돈에 해당하는 아미노산을 찾는다.
개념 5

멘델의 분리 법칙

① 핵심 개념
한 쌍의 대립유전자는 생식세포 형성 시 서로 분리되어 각각 다른 생식세포로 들어간다.
② 개념 상세 설명
  1. 멘델은 완두콩 실험으로 유전 법칙을 발견했다.
  2. 개체는 한 형질에 대해 두 개의 대립유전자를 가질 수 있다.
  3. 생식세포가 만들어질 때 대립유전자는 서로 분리된다.
  4. 자손은 부모로부터 각각 하나씩 대립유전자를 받는다.
  5. 이 법칙은 단일 유전자 형질의 유전을 설명한다.
③ 활용 예제
유전 확률 계산, 가계도 분석, 품종 개량, 유전 상담에 활용된다.
④ 파이썬 코드
# Aa x Aa 교배 결과
parents = ["A", "a"]
offspring = []

for p1 in parents:
    for p2 in parents:
        offspring.append(p1 + p2)

print(offspring)
⑤ 코드 상세 설명
  1. `parents = ['A', 'a']` : 한 부모가 만들 수 있는 생식세포이다.
  2. 중첩 `for`문은 두 부모의 생식세포 조합을 만든다.
  3. `p1 + p2` : 자손의 유전자형이다.
  4. `offspring.append(...)` : 가능한 자손 유전자형을 저장한다.
  5. 결과는 AA, Aa, aA, aa 조합이다.
개념 6

자연선택

① 핵심 개념
환경에 더 잘 적응한 형질을 가진 개체가 더 많이 생존하고 번식하여 그 형질의 빈도가 증가한다.
② 개념 상세 설명
  1. 개체들 사이에는 유전적 변이가 존재한다.
  2. 환경은 모든 개체에게 같은 생존 기회를 주지 않는다.
  3. 유리한 형질을 가진 개체는 더 잘 살아남고 번식할 가능성이 높다.
  4. 그 형질은 다음 세대에 더 자주 전달된다.
  5. 오랜 시간이 지나면 집단의 형질 빈도가 변화하여 진화가 일어난다.
③ 활용 예제
항생제 내성, 산업 melanism, 품종 개량, 진화 시뮬레이션에 활용된다.
④ 파이썬 코드
# 유리한 형질 빈도 증가 단순 모델
freq = 0.1
selection_advantage = 0.05

for generation in range(10):
    freq += selection_advantage * freq * (1 - freq)
    print(generation + 1, round(freq, 3))
⑤ 코드 상세 설명
  1. `freq` : 유리한 형질의 초기 빈도이다.
  2. `selection_advantage` : 선택 이점이다.
  3. `freq * (1 - freq)` : 빈도가 0이나 1에 가까우면 변화가 작아지는 효과를 준다.
  4. 반복할수록 유리한 형질의 빈도가 증가한다.
  5. 자연선택의 단순 모델이다.
개념 7

항상성

① 핵심 개념
생물은 체온, 혈당, pH 같은 내부 환경을 일정 범위로 유지한다.
② 개념 상세 설명
  1. 항상성은 생명체가 안정된 내부 상태를 유지하는 능력이다.
  2. 체온 조절, 혈당 조절, 삼투압 조절이 대표적인 예이다.
  3. 항상성은 주로 음성 피드백으로 유지된다.
  4. 내부 상태가 기준값에서 벗어나면 이를 되돌리는 반응이 일어난다.
  5. 항상성이 무너지면 질병이나 생명 활동 이상이 발생할 수 있다.
③ 활용 예제
체온 조절, 당뇨병 이해, 운동 생리, 의학 진단에 활용된다.
④ 파이썬 코드
# 혈당 음성 피드백 단순 모델
blood_sugar = 150
target = 100

for step in range(5):
    correction = 0.3 * (blood_sugar - target)
    blood_sugar -= correction
    print(step + 1, round(blood_sugar, 1))
⑤ 코드 상세 설명
  1. `blood_sugar` : 현재 혈당이다.
  2. `target` : 정상 목표 혈당이다.
  3. `blood_sugar - target` : 목표값과의 차이이다.
  4. `correction` : 몸이 혈당을 낮추는 보정량이다.
  5. 반복할수록 혈당이 목표값에 가까워진다.
개념 8

효소와 활성화 에너지

① 핵심 개념
효소는 반응의 활성화 에너지 \(E_a\)를 낮추어 생화학 반응 속도를 증가시킨다.
② 개념 상세 설명
  1. 효소는 생체 촉매이다.
  2. 촉매는 반응 전후에 소모되지 않는다.
  3. 효소는 특정 기질과 결합하여 반응이 쉽게 일어나도록 돕는다.
  4. 효소는 활성화 에너지를 낮춘다.
  5. 온도와 pH가 적절하지 않으면 효소 구조가 변해 기능이 떨어질 수 있다.
③ 활용 예제
소화, 대사 반응, 세제 효소, 식품 산업, 의약품 개발에 활용된다.
④ 파이썬 코드
# 활성화 에너지 감소에 따른 반응속도 비교
import math

Ea_without = 80
Ea_with = 50
R = 8.314
T = 300

rate_ratio = math.exp(-(Ea_with - Ea_without) / (R * T))

print("효소 사용 시 상대 속도:", rate_ratio)
⑤ 코드 상세 설명
  1. `Ea_without` : 효소가 없을 때 활성화 에너지이다.
  2. `Ea_with` : 효소가 있을 때 활성화 에너지이다.
  3. `R` : 기체상수이다.
  4. `T` : 절대온도이다.
  5. 활성화 에너지가 낮아지면 반응속도는 증가한다.
개념 9

광합성

① 핵심 개념
식물은 빛에너지를 이용해 이산화탄소와 물로부터 포도당과 산소를 만든다. \[6CO_2+6H_2O\to C_6H_{12}O_6+6O_2\]
② 개념 상세 설명
  1. 광합성은 엽록체에서 일어난다.
  2. 빛 반응에서는 빛에너지를 이용해 ATP와 NADPH를 만든다.
  3. 암반응에서는 이산화탄소를 고정해 포도당을 합성한다.
  4. 광합성은 지구 대기의 산소를 공급하는 핵심 과정이다.
  5. 생태계의 에너지 흐름은 광합성에서 시작된다.
③ 활용 예제
농업 생산성, 탄소 순환, 기후 변화, 식물 생장 연구에 활용된다.
④ 파이썬 코드
# 빛 세기에 따른 광합성량 단순 모델
light = 80
max_rate = 100

rate = max_rate * light / (light + 50)

print("광합성 속도:", rate)
⑤ 코드 상세 설명
  1. `light` : 빛의 세기이다.
  2. `max_rate` : 최대 광합성 속도이다.
  3. `light / (light + 50)` : 빛이 증가할수록 포화되는 효과를 나타낸다.
  4. `rate` : 단순화한 광합성 속도이다.
  5. 실제 광합성은 온도, 물, 이산화탄소 농도에도 영향을 받는다.
개념 10

세포호흡

① 핵심 개념
세포는 포도당을 분해하여 ATP를 생산한다. \[C_6H_{12}O_6+6O_2\to6CO_2+6H_2O+\mathrm{ATP}\]
② 개념 상세 설명
  1. 세포호흡은 세포가 에너지를 얻는 과정이다.
  2. 주로 미토콘드리아에서 일어난다.
  3. 포도당은 해당과정, TCA 회로, 전자전달계를 거쳐 분해된다.
  4. 산소가 충분하면 많은 ATP가 생성된다.
  5. ATP는 세포 활동에 사용되는 에너지 화폐이다.
③ 활용 예제
운동 생리, 대사 질환, 에너지 흐름, 생물학 실험에 활용된다.
④ 파이썬 코드
# 포도당 분자 수에 따른 ATP 생산량 근사
glucose = 10
atp_per_glucose = 30

total_atp = glucose * atp_per_glucose

print("예상 ATP 생산량:", total_atp)
⑤ 코드 상세 설명
  1. `glucose` : 포도당 분자 수를 단순화한 값이다.
  2. `atp_per_glucose` : 포도당 1개당 ATP 생산량 근사값이다.
  3. `total_atp` : 전체 ATP 생산량이다.
  4. 산소 호흡은 포도당에서 많은 ATP를 얻는다.
개념 11

세포막과 선택적 투과성

① 핵심 개념
세포막은 인지질 이중층으로 이루어져 있으며 필요한 물질은 통과시키고 불필요한 물질은 제한한다.
② 개념 상세 설명
  1. 세포막은 세포 안과 밖을 구분한다.
  2. 인지질 이중층은 소수성 내부와 친수성 외부를 가진다.
  3. 작고 비극성인 물질은 비교적 쉽게 통과한다.
  4. 이온이나 큰 분자는 단백질 통로가 필요하다.
  5. 선택적 투과성은 세포 내부 환경 유지에 필수적이다.
③ 활용 예제
약물 흡수, 삼투 현상, 신경 신호, 물질 수송 이해에 활용된다.
④ 파이썬 코드
# 분자 크기와 전하에 따른 통과 가능성 단순 판단
size = "small"
charged = False

can_pass = (size == "small") and (not charged)

print("세포막 직접 통과 가능?", can_pass)
⑤ 코드 상세 설명
  1. `size` : 분자 크기이다.
  2. `charged` : 전하가 있는지 여부이다.
  3. 작고 전하가 없는 분자는 막을 통과하기 쉽다.
  4. `and`는 두 조건이 모두 참인지 확인한다.
  5. `can_pass`는 직접 통과 가능성을 나타낸다.

삼투

① 핵심 개념
물은 반투과성 막을 사이에 두고 용질 농도가 낮은 곳에서 높은 곳으로 이동한다.
② 개념 상세 설명
  1. 삼투는 물 분자의 확산 현상이다.
  2. 반투과성 막은 물은 통과시키지만 일부 용질은 통과시키지 않는다.
  3. 용질 농도가 높은 쪽은 물을 끌어당긴다.
  4. 식물 세포의 팽압과 동물 세포의 부피 변화에 중요하다.
  5. 삼투압 차이가 크면 세포가 터지거나 쭈그러질 수 있다.
③ 활용 예제
식물 팽압, 수액 이동, 혈액 삼투압, 신장 기능, 식품 저장에 활용된다.
④ 파이썬 코드
# 농도 차이에 따른 물 이동 방향
inside = 0.3
outside = 0.1

if inside > outside:
    direction = "세포 안으로 물 이동"
elif inside < outside:
    direction = "세포 밖으로 물 이동"
else:
    direction = "순이동 없음"

print(direction)
⑤ 코드 상세 설명
  1. `inside` : 세포 안 용질 농도이다.
  2. `outside` : 세포 밖 용질 농도이다.
  3. 농도가 높은 쪽으로 물이 이동한다.
  4. 조건문으로 물 이동 방향을 판단한다.
  5. 농도가 같으면 물의 순이동은 없다.

생태계 에너지 흐름

① 핵심 개념

② 개념 상세 설명
  1. 생태계 에너지의 출발점은 주로 태양이다.
  2. 생산자는 광합성으로 에너지를 저장한다.
  3. 초식동물은 생산자를 먹고 에너지를 얻는다.
  4. 육식동물은 다른 소비자를 먹고 에너지를 얻는다.
  5. 영양 단계가 올라갈수록 전달되는 에너지는 줄어든다.
③ 활용 예제
먹이사슬, 생태 피라미드, 생물량 분석, 환경 교육에 활용된다.

# 10% 에너지 전달 법칙

⑤ 코드 상세 설명
  1. `energy` : 생산자 단계의 에너지이다.
  2. `for level in range(1, 5)` : 네 영양 단계를 출력한다.
  3. `energy *= 0.1` : 다음 단계로 약 10%만 전달된다고 가정한다.
  4. 영양 단계가 높아질수록 에너지가 감소한다.

개체군 성장

① 핵심 개념
자원이 충분할 때 개체군은 지수적으로 증가할 수 있으며, 제한된 환경에서는 로지스틱 성장에 가까워진다.
② 개념 상세 설명
  1. 개체군은 특정 지역에 사는 같은 종의 집단이다.
  2. 초기에는 자원이 충분하면 빠르게 증가한다.
  3. 환경수용력은 환경이 감당할 수 있는 최대 개체수이다.
  4. 개체수가 환경수용력에 가까워지면 성장률이 감소한다.
  5. 로지스틱 성장은 제한된 자원을 고려한 모델이다.
③ 활용 예제
생태 예측, 어업 관리, 감염병 확산, 보전생물학에 활용된다.
④ 파이썬 코드
# 로지스틱 성장 모델
N = 10
r = 0.5
K = 100

for t in range(10):
    N += r * N * (1 - N / K)
    print(t + 1, round(N, 2))
⑤ 코드 상세 설명
  1. `N` : 현재 개체수이다.
  2. `r` : 성장률이다.
  3. `K` : 환경수용력이다.
  4. `1 - N / K` : 개체수가 많아질수록 성장이 느려지는 효과이다.
  5. 로지스틱 모델은 제한된 환경의 성장을 나타낸다.

생물 다양성

① 핵심 개념
생물 다양성은 유전적 다양성, 종 다양성, 생태계 다양성을 포함한다.
② 개념 상세 설명
  1. 유전적 다양성은 같은 종 안의 유전자 차이이다.
  2. 종 다양성은 한 지역에 존재하는 종의 다양함이다.
  3. 생태계 다양성은 다양한 서식지와 생태계 유형을 뜻한다.
  4. 생물 다양성이 높으면 생태계 안정성이 커질 수 있다.
  5. 인간 활동은 서식지 파괴와 기후 변화로 다양성을 감소시킨다.
③ 활용 예제
보전생물학, 환경 영향 평가, 생태계 관리, 멸종위기종 보호에 활용된다.
④ 파이썬 코드
# 간단한 종 다양도 계산
species_counts = [50, 30, 20]
total = sum(species_counts)

proportions = [n / total for n in species_counts]
diversity = 1 - sum(p**2 for p in proportions)

print("다양도 지수:", diversity)
⑤ 코드 상세 설명
  1. `species_counts` : 각 종의 개체수이다.
  2. `total` : 전체 개체수이다.
  3. `proportions` : 각 종의 비율이다.
  4. `sum(p**2 for p in proportions)` : 한 종에 치우친 정도를 계산한다.
  5. `diversity`가 클수록 더 다양한 군집이다.

공생

① 핵심 개념
서로 다른 종이 가까이 살며 영향을 주고받는 관계를 공생이라고 한다.
② 개념 상세 설명
  1. 상리공생은 두 종 모두 이익을 얻는 관계이다.
  2. 편리공생은 한 종은 이익을 얻고 다른 종은 큰 영향이 없다.
  3. 기생은 한 종이 이익을 얻고 다른 종이 피해를 입는 관계이다.
  4. 공생 관계는 생태계 상호작용의 중요한 부분이다.
  5. 대표 예로 꽃과 곤충, 장내 미생물, 기생충 관계가 있다.
③ 활용 예제
생태계 상호작용, 미생물 연구, 농업, 감염병 이해에 활용된다.
④ 파이썬 코드
# 공생 관계 분류
effect_A = 1
effect_B = 1

if effect_A > 0 and effect_B > 0:
    relation = "상리공생"
elif effect_A > 0 and effect_B == 0:
    relation = "편리공생"
elif effect_A > 0 and effect_B < 0:
    relation = "기생"
else:
    relation = "기타 관계"

print(relation)
⑤ 코드 상세 설명
  1. `effect_A`, `effect_B` : 두 생물에게 미치는 영향을 숫자로 나타낸다.
  2. 양수는 이익, 0은 영향 없음, 음수는 피해이다.
  3. 둘 다 양수이면 상리공생이다.
  4. 한쪽만 이익이면 편리공생 또는 기생이 될 수 있다.
  5. 조건문으로 생물 관계를 분류한다.
개념 20

진화와 유전자 빈도

① 핵심 개념
진화는 세대에 걸쳐 집단의 유전자 빈도가 변화하는 과정이다.
② 개념 상세 설명
  1. 진화는 개체가 아니라 집단 수준에서 일어난다.
  2. 돌연변이는 새로운 유전 변이를 만든다.
  3. 자연선택은 유리한 형질의 빈도를 높일 수 있다.
  4. 유전적 부동은 우연에 의해 유전자 빈도가 변하는 현상이다.
  5. 이주도 집단의 유전자 빈도를 바꿀 수 있다.
③ 활용 예제
진화 연구, 집단유전학, 보전생물학, 질병 유전자 분석에 활용된다.
④ 파이썬 코드
# 유전자 빈도 변화 단순 모델
p = 0.4
fitness_A = 1.1
fitness_a = 1.0

for generation in range(5):
    p = (p * fitness_A) / (p * fitness_A + (1 - p) * fitness_a)
    print(generation + 1, round(p, 3))
⑤ 코드 상세 설명
  1. `p` : 특정 유전자의 초기 빈도이다.
  2. `fitness_A` : 해당 유전자의 적합도이다.
  3. `fitness_a` : 다른 유전자의 적합도이다.
  4. 적합도가 높으면 다음 세대 빈도가 증가한다.
  5. 반복문은 여러 세대의 변화를 보여 준다.
